# Apache Atlas 组件介绍

## 概述

Apache Atlas 是 Apache 软件基金会维护的开源元数据管理与数据治理框架。它为企业提供了一个集中化平台，用于在复杂、异构的数据生态系统中对数据资产进行编目、治理和发现。Atlas 帮助组织理解数据的来源、流向和用途，是构建可信数据环境的基石。

## 核心特性

- **元数据建模（Type-Entity 系统）**：提供灵活的类型系统，允许用户定义技术元数据和业务元数据的模型。类型的实例被称为"实体（Entity）"。
- **数据血缘追踪（Data Lineage）**：自动跟踪数据在各系统间的移动、转换和演变过程，提供直观的血缘关系可视化图谱。
- **分类与标签（Classification & Tagging）**：支持动态地为数据资产打上分类标签（如 PII、敏感数据、机密），标签可沿血缘链路自动传播，确保治理策略一致性。
- **搜索与发现（Search & Discovery）**：提供强大的搜索界面和领域特定查询语言（DSL），支持类 SQL 的复杂条件搜索。
- **安全集成**：与 Apache Ranger 深度整合，实现基于标签的细粒度访问控制与数据脱敏。
- **业务术语表（Business Glossary）**：支持创建层次化的业务词汇表，将技术元数据与业务定义对齐。

## 系统架构

Atlas 架构分为三层：

1. **核心层（Core）**
   - **Type System**：定义元数据对象及其关系的模型。
   - **Graph Engine**：使用 JanusGraph 图数据库存储元数据实体和关系，支持复杂的关联查询。
   - **存储与索引**：底层使用 HBase 存储图数据，使用 Solr（或 Elasticsearch）提供全文搜索索引。

2. **集成层（Ingest/Export）**
   - **Hooks/Bridges**：嵌入到 Hive、Spark、Sqoop 等组件中的钩子库，自动捕获元数据变更事件。
   - **Kafka**：作为通知服务总线，实现数据源与 Atlas 之间实时、松耦合的消息传递。
   - **REST API**：提供完整的 CRUD 接口，供外部系统程序化地管理元数据。

3. **应用层（Applications）**
   - **Admin UI**：Web 界面，支持搜索、浏览、标注和可视化数据血缘。
   - **Ranger 集成**：基于 Atlas 中的分类标签，在 Ranger 中执行安全策略。

## 适用场景

- 需要在多数据源环境中进行统一元数据管理的企业
- 需要满足数据合规审计要求，追踪数据来源与流向
- 数据湖/数据仓库中的数据发现与自助分析
- 与 Ranger 配合实现标签驱动的安全治理
